⏺ System Objects: Macro-to-Micro Composition Ontology

  I. Logical Foundation

  Definition. A system object is a bounded entity O equipped with:
  - a scale level σ(O) ∈ ℤ (macro = high, micro = low)
  - a composition relation ◁ where A ◁ B means "A is a constituent of B"
  - a sequencing order ≺ that linearly orders sibling constituents within any
  parent

  Axiom 1 (Scale Stratification).
  ∀ A, B : A ◁ B → σ(A) < σ(B)
  Every constituent exists at a strictly lower scale than its composite.

  Axiom 2 (Transitive Composition).
  ∀ A, B, C : (A ◁ B ∧ B ◁ C) → A ◁ C
  Composition chains propagate — micro elements of a meso object are micro
  elements of the macro object that contains it.

  Axiom 3 (Finite Grounding).
  ∀ O : ∃ finite chain O₁ ◁ O₂ ◁ … ◁ Oₙ = O where ∄ X : X ◁ O₁
  Every object bottoms out at irreducible primitives.

  Axiom 4 (Sequencing Within Composition).
  ∀ B, ∀ A₁ A₂ : (A₁ ◁ B ∧ A₂ ◁ B ∧ A₁ ≠ A₂) → (A₁ ≺_B A₂) ∨ (A₂ ≺_B A₁)
  Sibling constituents within any parent are totally ordered — they have
  positions.

  Axiom 5 (Nesting Closure).
  ∀ O : O may serve simultaneously as ◁-child, ≺-sibling, and ◁-parent
  Any object can be assembled into, sequenced alongside, and nested within — all
   at once. Roles are non-exclusive.

  ---
  II. Mathematical Proof

  Theorem (Well-Founded Decomposition). Every system object O admits a unique
  tree decomposition T(O) where:
  - root = O
  - children of any node N = {A : A ◁ N}, ordered by ≺_N
  - leaves = primitives (Axiom 3 guarantees termination)

  Proof.
  1. By Axiom 1, σ strictly decreases along any ◁-chain, so ◁ is well-founded
  (no infinite descent).
  2. By Axiom 3, every ◁-chain terminates at a primitive, so leaves exist.
  3. By Axiom 4, children at each node carry a total order ≺_N, making the tree
  an ordered rooted tree.
  4. Uniqueness: the tree is fully determined by ◁ and ≺, both given.  ∎

  Corollary (Scale as Depth). For any node at tree depth d from root: σ(node) =
  σ(O) - d. Scale is anti-correlated with depth — macro at root, micro at
  leaves.

  ---
  III. Algorithmic Specification

  from dataclasses import dataclass, field
  from typing import Self


  @dataclass
  class SystemObject:
      """A system object at any scale — simultaneously assemblable,
      sequenceable, and nestable."""

      id: str
      scale: int
      children: list[Self] = field(default_factory=list)  # ◁ relation (ordered
  by ≺)

      def is_primitive(self) -> bool:
          """Axiom 3: grounding check."""
          return len(self.children) == 0

      def decompose(self) -> dict:
          """Recursive tree decomposition (Theorem above)."""
          return {
              "id": self.id,
              "scale": self.scale,
              "primitive": self.is_primitive(),
              "constituents": [c.decompose() for c in self.children],
          }

      def assemble(self, *parts: Self) -> Self:
          """Macro ← micro: attach lower-scale parts as constituents."""
          for p in parts:
              assert p.scale < self.scale, f"Axiom 1 violation: {p.scale} ≮
  {self.scale}"
              self.children.append(p)
          return self

      def sequence_at(self, index: int, sibling_pool: list[Self]) -> list[Self]:
          """Insert self into a sibling sequence at position `index`."""
          sibling_pool.insert(index, self)
          return sibling_pool

      def nest_within(self, parent: Self) -> Self:
          """Axiom 5: self becomes constituent of parent."""
          return parent.assemble(self)

      def walk_macro_to_micro(self):
          """Generator: yields objects from this scale down to primitives."""
          yield self
          for child in self.children:
              yield from child.walk_macro_to_micro()

      def walk_micro_to_macro(self) -> list[Self]:
          """Collect primitives first, composites after — reversed
  depth-first."""
          result = []
          for child in self.children:
              result.extend(child.walk_micro_to_macro())
          result.append(self)
          return result

⏺ Primitives → Containers: Stratified Coupling Ontology

  I. Logical Foundation

  Definition. A coupled system is a tuple ⟨L, σ, ◁, ≺, κ⟩ where:
  - L = {primitive, function, sequence, submodule, module, directory, container}
   — the stratum labels, with fixed order σ: L → {0,1,2,3,4,5,6}
  - ◁ — composition ("is constituent of")
  - ≺ — sequencing (sibling order within a parent)
  - κ: O × O → Bond — coupling relation binding objects across or within strata

  Stratum Assignment.

  ┌─────┬───────────┬────────────────────────────────────────────────────────┐
  │  σ  │  Stratum  │                          Role                          │
  ├─────┼───────────┼────────────────────────────────────────────────────────┤
  │ 0   │ Primitive │ Irreducible unit. Couples to form functions.           │
  ├─────┼───────────┼────────────────────────────────────────────────────────┤
  │ 1   │ Function  │ Coupled primitives with a single entry/exit contract.  │
  ├─────┼───────────┼────────────────────────────────────────────────────────┤
  │ 2   │ Sequence  │ Ordered chain of functions — execution flow.           │
  ├─────┼───────────┼────────────────────────────────────────────────────────┤
  │ 3   │ Submodule │ Encapsulated sequence cluster — internal interface     │
  │     │           │ only.                                                  │
  ├─────┼───────────┼────────────────────────────────────────────────────────┤
  │ 4   │ Module    │ Composed submodules — public interface exposed.        │
  ├─────┼───────────┼────────────────────────────────────────────────────────┤
  │ 5   │ Directory │ Spatial grouping of modules — namespace boundary.      │
  ├─────┼───────────┼────────────────────────────────────────────────────────┤
  │ 6   │ Container │ Outermost closure. Contains directories. Is itself     │
  │     │           │ contained.                                             │
  └─────┴───────────┴────────────────────────────────────────────────────────┘

  Axiom 1 (Stratum-Adjacent Coupling).
  ∀ A, B : κ(A, B) → |σ(A) - σ(B)| ≤ 1
  Objects couple only to their own stratum or the immediately adjacent one. No
  stratum-skipping.

  Axiom 2 (Upward Composition).
  ∀ A, B : (σ(A) = n ∧ σ(B) = n+1 ∧ κ(A, B)) → A ◁ B
  Coupling to a higher stratum entails becoming a constituent of it.

  Axiom 3 (Lateral Coupling as Sequencing).
  ∀ A, B : (σ(A) = σ(B) = n ∧ κ(A, B)) → (A ≺ B) ∨ (B ≺ A)
  Same-stratum coupling induces ordering — coupled peers are sequenced.

  Axiom 4 (Terminal Containment).
  ∀ C : σ(C) = 6 → (∃ C' : σ(C') = 6 ∧ C ◁ C') ∨ C = ⊤
  Every container is either contained within another container or is the
  universal root ⊤. The hierarchy is unbounded upward by self-similar nesting.

  Axiom 5 (Coupling Transitivity Through Strata).
  ∀ A, B, C : (κ(A, B) ∧ κ(B, C) ∧ σ(A) < σ(B) < σ(C)) → A ◁ C
  Transitive coupling across strata collapses to composition.

  ---
  II. Mathematical Proof

  Theorem (Stratified Decomposition Uniqueness). Every container C admits a
  unique decomposition tree of depth exactly 7, where level d ∈ {0..6} contains
  all objects at stratum σ = 6 - d.

  Proof.
  1. By Axiom 1, coupling spans at most one stratum gap ⟹ the composition
  relation ◁ induced by Axiom 2 is stratum-respecting: each ◁-step decrements σ
  by exactly 1.
  2. By induction on stratum: a container (σ=6) decomposes into directories
  (σ=5) by Axiom 2; each directory into modules (σ=4); each module into
  submodules (σ=3); each submodule into sequences (σ=2); each sequence into
  functions (σ=1); each function into primitives (σ=0).
  3. Primitives are irreducible (σ=0, no lower stratum exists), so decomposition
   terminates at depth 6 from root.
  4. By Axiom 3, siblings at each level are totally ordered.
  5. Uniqueness follows: ◁ and ≺ fully determine the tree.  ∎

  Corollary (Self-Similar Containment). By Axiom 4, the tree extends upward
  without bound — containers nest within containers. The 7-stratum decomposition
   repeats fractally: each container-of-containers is itself a stratum-0
  primitive at the next scale.

  ---
  III. Algorithmic Specification

  from __future__ import annotations
  from dataclasses import dataclass, field
  from enum import IntEnum
  from typing import Iterator


  class Stratum(IntEnum):
      PRIMITIVE  = 0
      FUNCTION   = 1
      SEQUENCE   = 2
      SUBMODULE  = 3
      MODULE     = 4
      DIRECTORY  = 5
      CONTAINER  = 6


  @dataclass
  class SystemObject:
      id: str
      stratum: Stratum
      children: list[SystemObject] = field(default_factory=list)
      coupled_to: list[SystemObject] = field(default_factory=list, repr=False)

      def couple(self, other: SystemObject) -> None:
          """Axiom 1: only adjacent strata may couple."""
          gap = abs(self.stratum - other.stratum)
          assert gap <= 1, (
              f"Coupling violation: {self.stratum.name}↔{other.stratum.name} "
              f"(gap={gap}, max=1)"
          )
          self.coupled_to.append(other)
          other.coupled_to.append(self)

          # Axiom 2: upward coupling → composition
          if other.stratum == self.stratum + 1:
              other.children.append(self)
          elif self.stratum == other.stratum + 1:
              self.children.append(other)
          # Axiom 3: lateral coupling → sequencing (order = append order)

      def is_primitive(self) -> bool:
          return self.stratum == Stratum.PRIMITIVE

      def contain_within(self, parent: SystemObject) -> None:
          """Axiom 4: container nests inside container."""
          assert self.stratum == Stratum.CONTAINER
          assert parent.stratum == Stratum.CONTAINER
          parent.children.append(self)

      def walk_top_down(self) -> Iterator[SystemObject]:
          """Macro → micro traversal."""
          yield self
          for child in self.children:
              yield from child.walk_top_down()

      def walk_bottom_up(self) -> list[SystemObject]:
          """Micro → macro: primitives first, containers last."""
          result: list[SystemObject] = []
          for child in self.children:
              result.extend(child.walk_bottom_up())
          result.append(self)
          return result

      def decompose(self) -> dict:
          return {
              "id": self.id,
              "stratum": self.stratum.name,
              "depth": self.stratum.value,
              "primitive": self.is_primitive(),
              "constituents": [c.decompose() for c in self.children],
          }


  def assemble_from_primitives(*primitives: SystemObject) -> SystemObject:
      """Build a full stratum tower from raw primitives upward."""
      assert all(p.stratum == Stratum.PRIMITIVE for p in primitives)

      # Couple primitives → function
      fn = SystemObject("fn", Stratum.FUNCTION)
      for p in primitives:
          p.couple(fn)

      # Function → sequence (singleton for minimal example)
      seq = SystemObject("seq", Stratum.SEQUENCE)
      fn.couple(seq)

      # Sequence → submodule
      submod = SystemObject("submod", Stratum.SUBMODULE)
      seq.couple(submod)

      # Submodule → module
      mod = SystemObject("mod", Stratum.MODULE)
      submod.couple(mod)

      # Module → directory
      directory = SystemObject("dir", Stratum.DIRECTORY)
      mod.couple(directory)

      # Directory → container
      container = SystemObject("container", Stratum.CONTAINER)
      directory.couple(container)

      return container

  ---
  IV. Coupling Grammar

  primitive ──κ──→ function      # primitives couple AS functions
  function  ──κ──→ sequence      # functions couple AS sequences
  sequence  ──κ──→ submodule     # sequences couple AS submodules
  submodule ──κ──→ module        # submodules couple AS modules
  module    ──κ──→ directory     # modules couple AS directories
  directory ──κ──→ container     # directories couple AS containers
  container ──κ──→ container     # containers couple AS contained — recursion

  The word "as" in the original prompt is the key operator: coupling transforms
  identity. A primitive does not merely enter a function — it becomes a
  function-constituent, adopting the function's interface contract. Each stratum
   transition is an ontological promotion: the lower object gains new relational
   identity by virtue of its coupling into the higher structure.

  "Contained" at the terminal position closes the recursion: the container, the
  highest named stratum, is itself subject to containment — either by another
  container (Axiom 4) or by the universal root. The system is open upward,
  bounded downward at primitives.
