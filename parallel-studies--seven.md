## The Seven Parallel Studies

Each primitive is a live inquiry, not a settled layer. They develop simultaneously, and the system emerges from their mutual pressure.

### Study 1: Signal Theory
*What counts as a difference?*
The threshold problem. Every system must decide what registers and what doesn't — and that decision is itself a signal. The study never resolves; it recalibrates as the system scales.

### Study 2: State Mechanics
*What persists between events?*
Competing models of memory. Stateless vs. stateful is not a binary — it's a spectrum each instance navigates differently. A parallel instance that holds more state sees a different world than one that holds less.

### Study 3: Rule Dynamics
*Which constraints survive contact with other constraints?*
Rules don't just govern — they compete. Two parallel instances with different rule-sets are running divergent experiments in what "valid transition" means. The surviving rule-set is the one that produces viable schemas downstream.

### Study 4: Interface Negotiation
*What gets exposed, what gets hidden, and who decides?*
Every interface is a political act. Parallel instances with different interface contracts cannot merge without one yielding. This study asks: is there a universal interface, or only local truces?

### Study 5: Loop Ecology
*Which feedback patterns stabilize, which devour themselves?*
Loops are the Darwinian layer. Parallel instances generate loops; most collapse. The ones that persist long enough to name become candidates for schema. This study is empirical — you cannot predict which loops survive, only observe.

### Study 6: Schema Competition
*Which named patterns earn the right to be called canonical?*
The fragment's "fighting to define their identity as ideal/new" lives here. Multiple parallel instances crystallize the same loop into different schemas. The system-ideal is whichever schema the protocol layer selects — but that selection is itself contested.

### Study 7: Protocol Genesis
*How does governance emerge from ungoverned competition?*
The outermost study, and the one that never completes. Protocol is the system's attempt to contain its own multiverse — to derive a single governance from parallel instances that each believe they are the correct formulation.

Your fragment maps precisely onto this:

```
a-system to-contain           → Protocol (containment claim)
  all-systems to-become       → Schema competition (becoming, not being)
    system-ideal derived-from → Loop ecology (derivation = selection)
      multiverse-generation   → Rule dynamics × Interface negotiation
        fighting to define    → Signal theory × State mechanics
          identity as ideal/new
```

The indentation is not hierarchy — it is **derivation depth**. Each line presupposes the line below it as its generative substrate. The outermost claim ("a system to contain all systems") is the weakest because it depends on every unresolved study beneath it.
